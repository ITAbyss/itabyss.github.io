慢『SQL』分析并且邮件发送

配置慢『SQL』

查看配置文件位置

find / -name my.cnf

编辑『my.cnf』，增加慢『sql』日志配置，如果没有，就新建。

vi /etc/my.cnf
[mysqld]
#============================================
# 设置1为开启
slow_query_log = 1

# 获取超过1秒以上的慢sql
long_query_time = 1

# 生成log文件的位置
slow_query_log_file = /tmp/mysqlslow.log

修改完配置后，记得重启『mysql』

/etc/init.d/mysqld restart

进入『mysql』查看是否生效

mysql -u [user] -pba
Enter password:
# 输入下列『sql』命令
show variables like '%slow_query_log%';

# Value出现『ON』说明已经开启，第二行是慢sql生成的路径

| Variable_name       | Value              |
+---------------------+--------------------+

| slow_query_log_file | /tmp/mysqlslow.log |
+---------------------+--------------------+
2 rows in set (0.00 sec)

慢『sql』分析

下载分析工具

mkdir ~/slow
cd ~/slow

wget https://raw.githubusercontent.com/percona/percona-toolkit/3.0/bin/pt-query-digest

给文件赋予权限

chmod +x pt-query-digest

下载『perl』依赖包

yum install perl-DBI -y
yum install perl-DBD-MySQL -y
yum install perl-Time-HiRes -y
yum install perl-IO-Socket-SSL -y

邮件服务

安装邮件服务

yum install sendmail -y
yum install mailx -y

启动邮件服务

service sendmail restart

测试邮件服务

mail -s "Test mail" maoweiwei@wywk.com < /etc/passwd

按上述操作，有可能发送失败，因为linux邮件服务器是默认用root用户来发送，很多邮件服务商会干掉这种root用户，避免邮件攻击，更好的方式是设置一个发送邮件是用户，然后在进行发送邮件

配置邮件

vi /etc/mail.rc
set from=ittestlog@test.cn
set smtp=smtp.exmail.qq.com
set smtp-auth-user=ittestlog@test.cn
set smtp-auth-password=xxxxxx
set smtp-auth=login

保存并重启

service restart sendmail

测试邮件发送

mail -s "Test mail" maoweiwei@wywk.com < /etc/passwd

编辑定时任务脚本

vi slow.sh
#! /bin/sh

# 本地IP
LOCALIP=$(/sbin/ip addr | grep inet | grep -v "127" | grep -v "inet6" | awk '{ print $2; }' | sed 's/\/.*$//')

# 报告文件名
REPORTNAME="/tmp/report.html"

# 慢sql文件
SLOWFILE="/tmp/mysqlslow.log"

# 空文件
NULL="/dev/null"

# 邮件
MAILTITLE="ypos mysql slow query --- ${LOCALIP}"
MAILFROM="sql@wywk.cn"

# 收件人
AddrList=(
        maoweiwei@wywk.cn
        heshaoliang@wywk.cn
)


# 写入模板
cat ${NULL} > ${REPORTNAME}

(
cat << EOF
<!DOCTYPE html>
<html>
<head>
<title></title>
</head>
<body>
<pre>
EOF
) >> ${REPORTNAME}

# 分析日志
~/slow/pt-query-digest ${SLOWFILE} >> ${REPORTNAME}

(
cat << EOF
</pre>
</body>
</html>
EOF
) >> ${REPORTNAME}

# 发送邮件
for each in ${AddrList[*]}
do
        mail -s "$(echo -e "${MAILTITLE}\nContent-Type: text/html; charset=utf-8")" ${each} < ${REPORTNAME}
done

# 清理日志
cat ${NULL} > ${SLOWFILE}

保存至文件夹，要保证同级目录下有『pt-query-digest』，给文件加执行权限

chmod +x slow.sh

目录结构应该是如下

slow/
├── pt-query-digest
└── slow.sh

测试脚本

sh slow.sh

编辑定时任务

配置每天 0 点 0 分发送报告邮件

crontab -e
0 0 * * * /bin/sh ~/slow/slow.sh >> /tmp/cron.log
