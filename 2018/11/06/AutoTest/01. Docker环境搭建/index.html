测试环境搭建

Docker主机

Host User Password
192.168.15.160 root 20DFull160

部署「mysql」

docker run -d --name yytest_mysql -p 33306:3306 -e MYSQL_ROOT_PASSWORD=root mysql:5.7

部署「redis」

docker run -d --name yytest_redis -p 36379:6379 redis:3.2 --requirepass "wangyu"

部署「portainer」docker管理工具

docker volume create portainer_data

docker run -d -p 9000:9000 --name portainer --restart always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer

访问

http://192.168.15.160:9000

制作可以「ssh」登录的环境

下载「CentOS」

docker pull centos:6.8

启动并且登录容器

docker run -it --name=yytest centos:6.8 /bin/bash

安装『sshd』依赖

yum -y install openssh-server
yum -y install openssh-clients

启动『sshd』服务，会报如下的错误，大致意识是要个什么『key』

# 守护启动服务
/usr/sbin/sshd -D

Could not load host key: /etc/ssh/ssh_host_rsa_key
Could not load host key: /etc/ssh/ssh_host_dsa_key
sshd: no hostkeys available -- exiting.

既然要『key』，就给它『key』，执行以下命令解决报错

ssh-keygen -t rsa -f /etc/ssh/ssh_host_rsa_key -N ""
ssh-keygen -t dsa -f /etc/ssh/ssh_host_dsa_key -N ""

再次启动服务，这是是守护模式启动，没有报错，就说明启动成功了。直接『Ctrl + C』开始配置登录密码

/usr/sbin/sshd -D

修改密码

passwd root

Changing password for user root.
New password:

设置完成后，退出容器，将容器保存成镜像

exit

查看容器『id』

docker ps -a

CONTAINER ID    IMAGE       COMMAND       CREATED              ... 
476efa1fd8e9    centos      "/bin/bash"   23 minutes ago       ...
...

将容器创建成为新的镜像，第一个参数是上面获取的容器『id』，第二个参数是取名『yytest-ssh』

docker commit 476efa1fd8e9 yytest_ssh

启动容器，将『22』远程连接的端口开放

docker run -d --name=yytest_ybill -p 30022:22 -p 38080:8080 yytest_ssh /usr/sbin/sshd -D

安装查看 ip 的软件

yum -y install initscripts

部署测试项目

先登录自己的 linux

ssh root@192.168.15.160 -P 30022

添加 key 鉴权，以后可以不用输入密码拉包

ssh-keygen -t rsa

查看鉴权

cat ~/.ssh/id_rsa.pub

拉 jdk

scp -r root@192.168.15.221:/usr/local/java /usr/local/java

添加 jdk 环境变量

vi /etc/profile

export JAVA_HOME=/usr/local/java/jdk1.8.0_172
export PATH=$JAVA_HOME/bin:$PATH
export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jarexport JRE_HOME=$JAVA_HOME/jre

使环境变量生效

source /etc/profile

拉包部署

scp -r root@192.168.15.221:/data/yBill /data/yBill

修改配置文件

cp /data/yBill/conf/application.yml /data/application-autotest.yml
cp /data/yBill/conf/application-test.yml /data/application-autotest-test.yml
cp /data/yBill/restart.sh /data/deploy.sh

JMeter 脚本编写

需要注意以下几点

  1. 尽量在一个线程组中写自动化测试用例
  2. 如果一定要分线程的话，需要在TestPlan中，勾选 独立运行每个线程组
  3. 在接口测试之前，尽量造自己的数据，以及清理上次测试的残留数据
    1. 数据初始化
    2. 测试用例
    3. 验证断言

善用参数化

  1. 参数会被覆盖
  2. 参数可以每次迭代更新
  3. 全局参数与局部参数

善用事物控制器

  1. 可以组合测试用例
  2. 可以让自己的逻辑更加清楚
  3. 命名规范。调试时，可以让自己更快的知道哪个地方错了

善用等待时间

  1. 单个等待时间与线程组等待时间

git 管理

  1. 下载
  2. 提交
  3. 上传

Linux 下运行脚本

AutomaticTest
├── build.py
├── jmeter4
├── result
└── testcase
名称 描述
build.py 用来执行自动化测试的脚本
jmeter4 jmeter版本
result 测试后生产结果的目录
testcase 测试用例的目录

build.py 的用处

  1. 清理上一次执行的自动化残留
  2. 更新测试用例
  3. 更新jmeter插件
  4. 遍历测试用例目录，获取测试用例文件
  5. 使用更新后的jmeter执行测试用例，将结果保存resut目录下
  6. 分析jtl文件，生产html报告
  7. 发送邮件
