2. JMeter 目录结构与继承关系

目录结构

『JMeter』 是由协议与功能区分目录的，这样的好处就是在 『JMeter』开发新的组件时，只需要编译新开发的组件，无需编译整个应用程序项目了。

源码编译可以看上一篇 Blog 『 ( 二次开发_1 ) IDEA 编译 JMeter 4.0 』

![image-20180908142028310](2. JMeter 目的结构与继承关系/image-20180908142028310.png)

顶级目录 描述
backups 备份目录，在运行脚本时，定时给你保存脚本，『JMeter 4.0』新特性
bin 运行目录，其中包含『jmeter』运行脚本，以及核心程序『ApacheJMeter.jar』
build 构建目录，在使用『Ant』编译后，这个目录会下会出现很多编译过的『.class』文件
docs 文档目录，源码编译后，这个目录没有什么东西
extras 扩展目录，这个目录在自动化测试时，可以用来优化测试报告
lib 引用目录，编译前是空的，编译后会有源程序的各种『jar』
licenese 规范目录，存放一些开源规范
printable_docs 文档目录，可以打印出来看的，有正规的格式很详细的文档
res 资源目录，存放着各个核心模块的『maven - pom』文件
src 源码目录，存放在『JMeter』的源码
test 测试目录，『JMeter』单元测试相关源码
xdocs 配置目录，主要存放『xml』文档，JMeter从『xml』生成相关文档

![image-20180908142144407](2. JMeter 目的结构与继承关系/image-20180908142144407.png)

lib 描述
api API规范目录，里面的东西是编译时所需要的
doc 此目录用于文档生成的『jar』，JMeter运行时，不需要它们
ext 包含『JMeter』的核心『jar』文件与协议。由于『JMeter』的包很多，『JMeter』在启动时会优先读取这个目录下的『jar』，来提升启动的速度。也可以存放第三方的『jar』
junit 用于启动单元测试的『jar』
opt 包含可选的『jar』文件，『JMeter』可以额外使用的功能，这个目录在『Ant』构建路径中

![image-20180908162743489](2. JMeter 目的结构与继承关系/image-20180908162743489.png)

src 描述
components 组件模块，包括断言、前置处理器、后置处理器、debug等
core 核心模块，『JMeter』核心代码框架，包括启动代码『NewDriver.java』
examples 示例代码，二次开发的例子
functions 函数模块，各种原生函数的实现
jorphan 工具模块，实用工具类，以及常见的实用函数
junit 测试模块，一些关于源码的测试代码
protocol 协议模块，包含各种协议的代码
protocol / ftp 协议模块下，ftp 组件源码，用来进行传输文件的测试
protocol / http 协议模块下，http 组件源码，用来访问 web 服务器进行测试
protocol / java 协议模块下，java 组件源码，可以二次开发一些 java 请求
protocol / jdbc 协议模块下，jdbc 组件源码，可以对数据库进行测试
protocol / jms 协议模块下，jms 组件源码，为负载测试 jms 消息传递服务进行测试
protocol / ldap 协议模块下，ldap 组件源码，可以对目录访问协议进行测试
protocol / mail 协议模块下，mail 组件源码，可以对邮件服务进行测试
protocol / mongodb 协议模块下，mongodb 组件源码，可以对 mongodb 进行测试
protocol / native 协议模块下，native 组件源码，可以对 native 协议进行测试
protocol / tcp 协议模块下，tcp 组件源码，可以对 tcp 协议的应用进行测试

继承关系

继承类 描述
AbstractFunction 为所有函数提供的通用方法，自定义函数必须继承它
AbstractTestElement 这个类为『Gui』类提供逻辑的代码，有全局变量，迭代规则之类的逻辑
AbstractPreProcessorGui 前置处理器界面
AbstractPostProcessorGui 后制处理器界面
AbstractSamplerGui Sampler界面
AbstractAssertionGui 断言界面
AbstractControllerGui 逻辑控制器界面
AbstractTimerGui 定时器界面
AbstractThreadGroupGui 线程组界面

根据上表，这里可以说明一下，『AbstractTestElement』可以作为任何一个『Gui』的逻辑类。一般『Gui』用来界面展现，当点击个别『按钮』时，会调用继承『AbstractTestElement』类的逻辑方法。
